Introdução

O que é o Freela Radar

Central operacional de agentes de IA que transforma oportunidades freelancer em propostas vencedoras — rodando 100% local, no seu computador.

O Freela Radar reúne, em um app de desktop, todo o fluxo de quem vive de freelancing técnico:

🕷️ Raspar vagas→ 🛰️ Classificar por match→ 🤖 Gerar proposta com IA→ 📤 Enviar

Os módulos

🛰️ Radar

Dashboard das oportunidades, classificadas pelo % de match com as suas tags, com KPIs do dia.

🕷️ Scrapper

Raspagem real do Workana com Playwright: múltiplas páginas, descrição completa e log ao vivo.

🤖 Studio

Editor visual de agentes: prompts, modelo, temperatura, tools e avatar.

🔗 Pipeline

Execução em cadeia dos agentes (PRD → Arquitetura → Pitch) sobre as vagas escolhidas.

Onde os dados ficam

Sem backend, sem conta, sem nuvem: tudo em SQLite e arquivos JSON/Markdown dentro do seu workspace local.

Pronto para começar? Vá para o Getting Started.

Introdução

Getting Started

Do clone ao primeiro pitch gerado em poucos minutos.

Pré-requisitos

  • Node.js 18+
  • Claude Code CLI instalado e autenticado (necessário para os agentes)
  • Chromium do Playwright (usado pelo Scrapper — instalado no passo a passo)

Instalação rápida

git clone https://github.com/seu-usuario/freela-radar.git
cd freela-radar
npm install
npm run rebuild              # recompila o better-sqlite3 para o Electron
npx playwright install chromium
npm run dev
Primeiro start

Na primeira execução o app pede para configurar seu nome, o caminho do banco e a pasta de workspace.

Fluxo de uso

  1. Raspe as vagas

    Abra Scrapper, informe a URL do Workana e quantas páginas percorrer, e clique em Iniciar raspagem. Acompanhe o log em tempo real.

  2. Classifique

    No Radar, clique em Executar varredura agora. As vagas são lidas de freelas/ e ordenadas pelo match com as suas tags.

  3. Gere a proposta

    Selecione uma ou mais vagas e rode o pipeline de agentes. Os documentos saem em workspace/oportunidades/.

Tags monitoradas

Cadastre suas tags e pesos em Radar → Editar tags. O comportamento do match fica em Settings → Match Engine.

Como instalar

Passo a passo detalhado para rodar o Freela Radar em desenvolvimento.

1. Pré-requisitos

RequisitoVersãoPara quê
Node.js18+Runtime do Electron e do build
npm9+Gerenciador de pacotes
Claude Code CLI—Executa os agentes (claude -p)
Chromium (Playwright)—Navegador do Scrapper

Confirme o Claude CLI com:

claude --version

2. Clone & instale

git clone https://github.com/seu-usuario/freela-radar.git
cd freela-radar
npm install

3. Recompile os módulos nativos

O better-sqlite3 precisa ser recompilado para a versão do Electron:

npm run rebuild

4. Baixe o navegador do Playwright

npx playwright install chromium
Sem isso o Scrapper não abre o navegador

Caso o download falhe, o Scrapper tenta usar o Edge ou o Chrome instalados no sistema (configurável em Settings → Playwright → Canal do navegador).

5. Rode

npm run dev        # desenvolvimento (hot reload)
npm run build      # build de produção
npm run preview    # roda o build de produção

Conceitos

O que é um agente

Um agente é uma persona de IA configurável que recebe uma vaga e produz um artefato (PRD, arquitetura, pitch…). Pense nele como um boneco que ganha vida: você dá um nome, uma alma, uma missão e uma consciência — e ele entrega um trabalho pronto.

A analogia do Pinóquio 🪵✨

Para fixar as partes de um agente, imagine montar o Pinóquio. Cada pedaço do boneco vira uma parte do agente:

🪵O boneco esculpido e batizado por Geppetto
↔
Nome & Função
✨A centelha de vida da Fada Azul
↔
Soul prompt (alma)
⭐O desejo de Geppetto: "um filho de verdade"
↔
System prompt (missão)
🦗O Grilo Falante — a consciência
↔
Guardrails prompt
📦Como o Pinóquio vai entregar o que fez
↔
Output configuration
🛠️A madeira e as mãos do artesão
↔
Modelo de execução
⚙️As engrenagens internas do boneco
↔
Runtime JSON
Em uma frase

O nome diz quem ele é, a alma dá personalidade, a missão dá um norte, a consciência impõe limites, e o resultado é o menino de verdade que ele entrega.

O roadmap de um agente

Percorra as 7 paradas abaixo — é a ordem em que você monta um agente no Studio. Cada parada mostra a peça do Pinóquio, o campo real e um exemplo.

1

Nome & Função

name · description 🪵 O boneco batizado

A carteira de identidade do agente. O nome identifica; a função (descrição) diz para que ele serve. É o primeiro passo: Geppetto esculpe o boneco e decide o que ele será.

Exemplo
name: "PRD Agent" · description: "Transforma uma vaga em um Documento de Requisitos acionável."
2

Soul prompt — a alma

soul_prompt ✨ A Fada Azul dá vida

Define quem o agente é: personalidade, tom de voz, especialidade e valores. É a centelha que transforma um boneco de madeira em alguém com caráter. No prompt final vira o bloco # IDENTIDADE.

Exemplo
"Você é um Product Manager sênior, pragmático e obcecado por clareza. Escreve direto ao ponto, sem jargão."
3

System prompt — a missão

system_prompt ⭐ "Um filho de verdade"

O norte do agente: o objetivo central que justifica a existência dele. Assim como o desejo de Geppetto move toda a história, a missão guia cada decisão. Vira o bloco # MISSÃO.

Exemplo
"Sua missão é produzir um PRD completo e acionável a partir da vaga recebida."
4

Guardrails prompt — a consciência

operational_prompt 🦗 O Grilo Falante

As regras e limites: o que pode, o que não pode, o passo a passo e o formato de trabalho. É o Grilo Falante sussurrando "não minta, não invente". Mantém o agente na linha. Vira o bloco # REGRAS OPERACIONAIS.

Exemplo
"Nunca invente requisitos — use apenas o que está na vaga. Estruture em: Contexto, Objetivos, Escopo, Critérios de aceite."
5

Output configuration — o resultado

output_format 📦 Como ele entrega

Como o agente entrega o que fez. O mesmo trabalho pode ser entregue como markdown, JSON ou checklist; é isso que define a estrutura e a extensão do arquivo salvo. Vira o bloco # FORMATO DE SAÍDA.

output_formatVira o arquivo
markdown · report · checklist.md
json.json
6

Modelo de execução — o "cérebro"

model + parâmetros 🛠️ Madeira & artesão

Do que o agente é feito e quão capaz ele é. Escolhe o modelo do Claude e os parâmetros que regulam a execução — a habilidade do artesão e a qualidade da madeira.

ParâmetroControlaValores
modelModelo do Claudesonnet · opus · haiku
temperatureCriatividade × consistência0.0 – 1.0
max_tokensTamanho máx. da respostaex.: 12000
effort_levelProfundidade de raciocíniolow · medium · high · maximum
autonomy_levelQuanto decide sozinhomanual · semi · autonomous · full
retries · timeout_secondsResiliência da execuçãotentativas · tempo limite
7

Runtime JSON — as engrenagens

runtime_config_json ⚙️ Engrenagens internas

Um JSON livre com ajustes finos de runtime, passados à execução. São as engrenagens escondidas do boneco: permitem casos especiais sem precisar criar novos campos.

{
  "stop_sequences": ["FIM"],
  "extra_context": "priorizar entregáveis com prazo curto",
  "verbosity": "concise"
}
Bônus · Ferramentas (tools)

Além das 7 paradas, um agente pode receber tools — como o cinzel e o martelo do artesão. São capacidades extras habilitáveis por agente, configuradas no Studio.

Como tudo vira o prompt final

Na execução, o app monta um único prompt juntando as partes do agente com os dados da vaga, nesta ordem:

# IDENTIDADE = soul_prompt # MISSÃO = system_prompt # REGRAS = guardrails # FORMATO = output_format 
# IDENTIDADE
{soul_prompt}

# MISSÃO
{system_prompt}

# REGRAS OPERACIONAIS
{operational_prompt}

# OPORTUNIDADE
- Título: ...
- Descrição: ...
- Orçamento: ...
- Tags: ...
- URL: ...

# FORMATO DE SAÍDA
{output_format}

# INSTRUÇÕES
Produza agora o artefato final, em português, pronto para envio ao cliente.

Agentes em cadeia (pipeline)

Vários agentes podem rodar em sequência, passando o resultado de um para o próximo (handoff). O pipeline padrão transforma uma vaga em uma proposta pronta:

📋 PRD→ 🏛️ Arquitetura→ 💬 Pitch
Onde edito tudo isso

No app: Studio (prompts, modelo, parâmetros, tools, avatar) e Pipeline (ordem e execução em lote). Cada execução salva o artefato em workspace/executions/.
